How to compile QGIS project

发表于 2019-01-14

官方依赖列表

Following a summary of the required dependencies for building:

Required build tools:

  • CMake >= 3.0.0
  • Flex >= 2.5.6
  • Bison >= 2.4

Required build dependencies:

  • Qt >= 5.3.0
  • Proj >= 4.4.x
  • GEOS >= 3.4
  • Sqlite3 >= 3.0.0
  • SpatiaLite >= 4.2.0
  • libspatialindex
  • GDAL/OGR >= 2.1
  • Qwt >= 5.0 & (< 6.1 with internal QwtPolar)
  • expat >= 1.95
  • QScintilla2
  • QCA
  • qtkeychain (>= 0.5)
  • libzip

Optional dependencies:

  • for GRASS providers and plugin - GRASS >= 7.0.0.
  • for georeferencer - GSL >= 1.8
  • for PostGIS support - PostgreSQL >= 8.0.x
  • for gps plugin - gpsbabel
  • for mapserver export and PyQGIS - Python >= 3.3
  • for python support - SIP >= 4.12, PyQt >= 5.3 must match Qt version, Qscintilla2
  • for qgis mapserver - FastCGI
  • for oracle provider - Oracle OCI library

Indirect dependencies:

Some proprietary formats (e.g., ECW and MrSid) supported by GDAL require
proprietary third party libraries. QGIS doesn’t need any of those itself to
build, but will only support those formats if GDAL is built accordingly. Refer
to http://gdal.org/formats_list.html ff. for instructions how to include
those formats in GDAL.

安装依赖

  • 安装 CMake
  • 安装 cygwin64, 勾选flex, bison

  • 双击 osgeo4w 安装包，选择本地安装源(skip表示不安装，keep表示已安装，版本号表示即将安装的版本)

    • 勾选qgis-rel-deps（二次开发包）
    • 或者手动安装所有的依赖，截图如下

  • ninja
    copy ninja.exe to C:\OSGeo4W64\bin\

更改脚本文件

  • 设置环境变量 cygwin64/bin
  • 修改QGIS\ms-windows\osgeo4w\msvc-env.cmd中OSGeo4W_ROOT变量
  • o4w-env.bat
  • 修改msvc-env.bat,
    set VS140COMNTOOLS=%PF86%\Microsoft Visual Studio\2017\Community\Common7\Tools
    call "%PF86%\Microsoft Visual Studio\2017\Community\VC\Auxiliary\Build\vcvarsall.bat" %VCARCH%
  • grass74.bat，关于OSGeoW_ROOT的路径

  • 在C:\OSGeo4W64目录下新建OSGeo4W-dev.bat文档并编辑

  • 运行F:\QGIS_Env\QGIS\ms-windows\osgeo4w路径下的configonly.bat

生成VS解决方案

  1. 由于使用的是VS2017，因此需要编辑configonly.bat，改为相应的 VS 版本（Visual Studio 15 2017）
  2. 编辑 package-nightly.bat，以电脑中lib库所在路径为准

Q&A

  • 编译qgs_core模块报以下错误

    检查环境变量，设置PATHONHOME和PATHONPATH。重启VS

  • C2001: 常量中有换行符

    是由于代码文件为 UTF-8，而 VS2017 支持的是 UTF-8 with BOM的编码格式，问题是cl编译的时候，对utf-8格式支持不好（需要添加/source-charset:utf-8选项，默认是当作本地字符集的），对于带BOM标记的文件则没有问题。

    方法一：将报错的文件编码更改为 UTF-8 with BOM，但操作较复杂，需要改动文件太多。

    方法二：更改本机电脑 代码页设置，最为方便快捷，但鉴于会导致 CityServer 开发环境编译报错。可能需要来回切换系统设置。

    方法三：在项目中添加附加选项/utf-8，可在CMake配置时修改选项CMAKE_CXX_FLAGS
